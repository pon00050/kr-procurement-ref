# 물품목록번호 체계 — 16자리 구조·대분류 정의·식별번호 원칙

## 개요

물품목록번호는 물품을 기능·용도·성질에 따라 **대·중·소·세**로 분류하고, 각 물품에 고유번호를 부여하는 체계이다. 주민등록번호와 유사하게 개별 물품을 식별하는 역할을 한다.

> [!note] 왜 표준화인가?
> 표준화 이전에는 부처마다 다른 분류 체계를 사용해 동일 품목이 여러 이름과 번호로 중복 등록됐다. 이는 가격 비교를 불가능하게 만들고, 구매 이력 분석·시장 규모 파악·감사 추적을 어렵게 했다. [[나라장터-도입성과-기능|나라장터]]의 목록정보 기능이 실효성을 갖추려면 — 406만 개 품목을 통합 검색하고 낙찰 가격을 비교하려면 — 품목 간 1:1 대응이 가능한 단일 분류 체계가 전제조건이다. 현재 9,900개 물품분류(품명)와 512만 개 품목 등록 정보를 이 체계로 관리한다.

## 현행 규정

### 물품목록번호 구조 (16자리)

```
물품목록번호 (16자리)
├── 물품분류번호 — 품명 (8자리)
│   ├── 대분류 (Segment, 2자리)
│   ├── 중분류 (Family, 2자리)
│   ├── 소분류 (Class, 2자리)
│   └── 세분류 (Commodity, 2자리)
└── 물품식별번호 — 품목 (8자리)
```

> [!note] 왜 분류번호(8자리)에는 의미가 있고 식별번호(8자리)에는 없는가?
> **분류번호**는 비교·검색·분석을 위해 의미를 담아야 한다 — "41번대 = 사무용품"처럼 번호 자체가 카테고리를 나타내야 범주 검색과 낙찰가 비교가 가능하다. 이 체계는 UN 국제표준인 **UNSPSC**(United Nations Standard Products and Services Code)를 채택한 것으로, 한국 공공조달 품목이 국제 분류 체계와 호환된다.
> **식별번호**는 고유성만 필요하다 — 개별 제품을 구별하는 역할이므로 일련번호로 충분하다. 의미를 부여하면 제품 속성 변경 시마다 번호를 재설계해야 하는 문제가 생긴다. 식별번호는 **GTIN**(Global Trade Item Number) 체계를 수용해 국제 호환성을 확보했다.

> **세부품명번호(10자리)** = 물품분류번호(8) + 2자리 추가 → 세부용도·재질·형태에 따라 추가 분류

### 4단계 분류 정의

| 분류 단계 | 영문명 | 자릿수 | 정의 |
|-----------|--------|--------|------|
| **대분류** | Segment | 2자리 | **분석적 목적의 집합** |
| **중분류** | Family | 2자리 | 상호 관련 상품군 |
| **소분류** | Class | 2자리 | 사용·기능 유사 상품군 |
| **세분류** | Commodity | 2자리 | 수요 적합 제품군 |

### 물품식별번호 (8자리) 부여 원칙

- 개별 물품을 구별하기 위한 **유일 고유번호**
- **숫자 자체에는 의미가 없음** (단순 일련번호 개념)
- 동일 품목 = **하나의 고유 식별번호** 부여 (모델·세부규격 동일하면 동일 식별번호)
- 중복 부여 시: **최초 부여된 번호 사용** (나중 부여된 번호 삭제)

### 목록화 4가지 원칙

| 원칙 | 내용 |
|------|------|
| **통일성** | 분류 대상·목적이 동일하면 분류체계는 단일화 |
| **포괄성** | 현재 품목 및 신규 품목 모두 포괄 가능하도록 범위 사전 책정 |
| **상호배제성** | 하나의 품목이 둘 이상의 품명으로 분류될 수 없음 |
| **단순성** | 분류명만 보면 바로 상품 특성 확인 가능 |

> 4가지 원칙: 통일성·포괄성·상호배제성·단순성 (다양성은 해당 없음)

### 품목등록 요청 4가지 방법

**실무 순서:** 먼저 [[나라장터-도입성과-기능|나라장터]] 목록정보시스템에서 기존 품목을 검색한다. 이미 등록된 품목이 있으면 참조 등록(기존요청 참조·기존품목 참조)으로 빠르게 처리하고, 없는 경우에만 신규 등록(추천 분류등록)을 요청한다.

| 방법 | 내용 |
|------|------|
| **추천 분류등록** | 품명·속성값 직접 입력 후 검색하여 분류 선택 |
| **기존요청 참조등록** | 기존 요청 정보 불러와 수정 후 재요청 |
| **기존품목 참조등록** | 기존 등록 품목 불러와 수정 후 요청 |
| **일괄등록** | 동일 품명·다수 품목 → 엑셀 양식 작성 후 일괄 요청 |

### 행정 처리 소요일수

| 구분 | 소요일수 |
|------|---------|
| **품목등록** | **8일** |
| **품목변경** | **5일** |
| **품명등록** | **24일** (신산업상품은 **17일**) |

## 시험 출제 포인트

**Q13 출제 패턴:** "물품목록 품목등록 요청 방법의 종류"
- 4가지: 추천 분류등록, 기존요청 참조등록, 기존품목 참조등록, 일괄등록

**Q14 출제 패턴:** "물품분류번호 체계의 대분류 정의"
- **대분류(Segment)** = **분석적 목적의 집합** (암기)
- 중분류 = 상호 관련 상품군, 소분류 = 사용·기능 유사, 세분류 = 수요 적합

**Q15 출제 패턴:** "물품식별번호의 숫자 의미 및 부여 원칙"
- **식별번호 숫자 자체에는 의미가 없음** → 핵심 정답
- 동일 품목 = 하나의 식별번호, 중복 시 최초 번호 사용

**Q5(3장) 출제 패턴:** "물품목록번호의 체계"
- 정답: **16자리** = 물품분류번호(8) + 물품식별번호(8)

**Q6(3장) 출제 패턴:** "목록화 원칙으로 옳지 않은 것"
- **다양성** → 오답 (단순성이 맞음)
- 유사어·제조사 명칭은 **유의어** (표준품명 아님)

**Q8(3장) 출제 패턴:** "품목등록 행정 처리 소요일수"
- **품목등록 = 8일**
